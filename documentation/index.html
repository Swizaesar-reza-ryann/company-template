Prerequisites: Make sure you have Node.js 18+ and npm installed on your system.

ğŸ¨ Customization Guide

Content Management

All website content is managed through the lib/data.ts file. This makes it easy to update text, links, and other content without touching component code.

Key Configuration Files:

  • lib/data.ts - All website content and configuration
  • tailwind.config.ts - Color scheme and design tokens
  • app/layout.tsx - SEO metadata and global settings

Changing Colors

Update the color scheme in tailwind.config.ts:

theme: {
  extend: {
    colors: {
      primary: {
        50: '#eff6ff',
        500: '#3b82f6',
        600: '#2563eb',
        700: '#1d4ed8',
        // ... more shades
      }
    }
  }
}

Updating Company Information

Edit the siteConfig object in lib/data.ts:

export const siteConfig = {
  name: 'Your Company Name',
  description: 'Your company description',
  url: 'https://your-domain.com',
  links: {
    email: 'contact@your-domain.com',
    phone: '+1 (555) 123-4567',
    address: 'Your address',
  },
  social: {
    twitter: 'https://twitter.com/your-handle',
    linkedin: 'https://linkedin.com/company/your-company',
    // ... other social links
  },
}

Adding New Pages

  1. Create a new folder in app/(site)/
  2. Add a page.tsx file
  3. Update navigation in lib/data.ts
  4. Add to sitemap in app/sitemap.ts

/>

ğŸ” SEO Configuration

Metadata Management

SEO is handled through Next.js Metadata API in lib/seo.ts:

  • Automatic Open Graph tags
  • Twitter Card support
  • Structured data (JSON-LD)
  • Robots meta configuration

Sitemap & Robots

Automatically generated sitemap and robots.txt:

  • /sitemap.xml - XML sitemap
  • /robots.txt - Search engine instructions

Page-Specific SEO

Each page can have custom metadata:

export const metadata: Metadata = generatePageMetadata(
  'Page Title',
  'Page description',
  '/page-path'
)

Performance Tips

  • Use priority prop for above-the-fold images
  • Optimize image sizes before uploading
  • Enable gzip compression on your server
  • Use CDN for static assets
